# Your Plan, Distilled

**Core concept**
- A robotics project-sharing hub — GitHub + MakerWorld, but built for robots
- Pros upload, beginners/kids download and build
- AI smooths the messy middle: documentation, setup guides, explanations

**Every post = 3 pillars**
- Code (paste, upload, or import from GitHub repo)
- Parts (checkbox list of ~30 common components; rare parts flagged/linked)
- 3D print files (CAD + STLs, with print instructions)
- Plus: name, description, instructions, and a "story/challenges" section

**Smart automation**
- Non-AI bot scans code → auto-suggests parts list (servo pins → servos, stepper libs → driver, etc.)
- AI reads code → drafts plain-English explanation + setup instructions for poster to approve/edit
- Template questions ("what does it do, how does it work") or free-text option
- Version history on updates (code, parts, files) — GitHub-style layering

**Accounts & posting**
- Username + password only; email optional
- Browse without account; post without account (get a "post password" for later edits)
- Captcha per post, not per account
- Simple word-filter auto-moderation kills most spam

**Community features**
- Remixes instead of edits — original stays untouched
- Forum/Q&A section (Reddit-style) + social feed for progress posts
- Requests section for kids to pitch robot ideas
- Discovery algorithm like MakerWorld

**The banger: open-source product replacers**
- Dedicated section: smart home, subscription-product alternatives
- Curated top 1–3 options per category (not a free-for-all)
- One-click "buy all parts" cart via Amazon/vendor links
- Vendors can sell pre-assembled kits → marketplace

**Education layer**
- Beginner explainers: 3.3V vs 5V, servo vs stepper, ESP32 vs Arduino vs Pi, power options
- Soldering alternatives section
- Goal: a kid finds out in minutes if their idea is possible + what it takes

---

# New Ideas Along the Same Lines

**Build tools**
- Interactive wiring diagram generator — select parts → auto-draws breadboard/pin diagram
- Pin conflict checker — flags when two parts want the same GPIO
- "Will it fit" power calculator — sums current draw, recommends battery/supply
- Web-based flasher (ESP Web Tools) — flash firmware from the browser, zero IDE install
- BOM cost estimator — live total price of a project before you commit

**Trust & quality signals**
- "I built this" verified badge — uploaders confirm with a photo/video of the working build
- Difficulty rating (beginner/intermediate/advanced) + estimated build time + estimated cost shown on every card
- "Tested on" tags: which board revisions/firmware versions people confirmed working
- Fail reports — structured "it didn't work because X" feedback that feeds back into instructions (your idea about common bugs, formalized)

**Onboarding funnels**
- "Idea checker" front-page tool: kid types "a robot that feeds my cat" → AI says possible/hard, lists parts, links similar projects
- Starter paths: curated project ladders (blinky LED → sensor → servo → full robot)
- School/classroom mode — teacher dashboards, class kits, bulk project assignments

**Content & discovery**
- "Part pages" — every sensor/motor gets its own page listing all projects using it (great SEO, great discovery)
- Weekly featured build + community challenges ("build something with one servo")
- Remix trees — visual family tree showing how projects evolved
- Video-first previews — 10-second clip of the robot working is the thumbnail

**Sustainability/monetization**
- Affiliate links on part lists (Amazon Associates) — natural revenue, sell nothing
- Kit vendor marketplace with small commission
- Sponsored part pages from manufacturers (clearly labeled)
- Optional supporter tier — no paywalls on content, ever

**Anti-enshittification insurance**
- Everything exportable — one-click download of your full post history
- Open API from day one — the anti-GitHub positioning is credibility; back it up
- Consider federating or open-sourcing the platform itself eventually

**Name-fit bonus**
- "Computronium" works — lean into the theme: turn dumb matter into smart matter

# Viral Mechanics

**The shareable moment**
- Every project page has a killer 10-sec video front and center — that's what gets shared on X/Reddit/TikTok
- One-click "share this build" cards: auto-generated image with robot pic + parts cost + build time ("This fish tank cleaner costs $23")
- The price tag IS the hook — "$23 robot" spreads; "cool robot" doesn't

**The "wait, I could do that" loop**
- Front-page Idea Checker: type any idea → instant "possible/hard/impossible" + cost + parts + similar projects
- That tool alone is shareable — people will screenshot its answers
- "Someone requested this" board → builder fulfills it → both get featured. Request-to-reality stories are viral gold

**Rage-bait (the good kind)**
- The subscription-replacer section is your viral engine: "Your smart blinds want $10/month? Build them for $18, one-time"
- Post side-by-side cost comparisons — these get shared angrily and often
- Time it to news cycles (company raises prices → your alternative page trends)

**Status for pros**
- Beautiful project pages — pros post where their work looks good (why people left GitHub READMEs for personal sites)
- Download/build counts, "X people built this" — real-world impact stats GitHub can't show
- Verified builds with photos = social proof pros crave

# The "What's Possible" Clarity System

This is your best idea. Make it structural, not just articles:

**Reality Check labels on every part**
- Every motor/part page states plainly: what it CAN do, what it CANNOT do, what it NEEDS
- Example — drone BLDC page: "❌ Cannot run slow or hold position without a $30+ FOC controller + encoder. Not for robot arms. ✅ Great for: fans, props, spinning fast"
- Servo page: "✅ Plugs straight into ESP32, no driver needed. ❌ Only rotates 180°, weak torque"
- Stepper page: "⚠️ Needs a driver board (A4988, $2). Cannot plug in directly"

**The Golden Rules page (pin it everywhere)**
- Motors never connect directly to an ESP32 — ever
- Servos: plug and play. Steppers: need driver. BLDC: need ESC (fast) or FOC (precise, $$$)
- Cheap + precise + strong: pick two
- 3.3V vs 5V will silently kill your board — check first

**Difficulty tiers on every project**
- 🟢 Plug & Play: order, print, flash, snap together (your farmer standard)
- 🟡 Some Assembly: soldering or wiring required
- 🔴 Advanced: custom electronics, tuning, debugging expected
- Filter the whole site by tier — beginners literally cannot stumble into 🔴 projects by accident

**The Idea Checker enforces this**
- "Robot arm with drone motors" → "Not viable as described: small BLDCs can't run slow/precise without FOC ($30+/motor). Use MG90S servos instead — here's 12 arm projects that do"
- It doesn't just say no — it redirects to the possible version

# How the AI Actually Works (Concrete)

**On upload (helps the pro)**
- Reads code → drafts: what it does, parts detected, pin connections, setup steps
- Poster approves/edits each section — AI drafts, human signs off
- Asks 3-5 questions only for what code can't reveal (assembly order, calibration, gotchas)
- Flags missing info: "Your code uses pin 13 for a relay but no relay is in your parts list — add it?"

**On download (helps the beginner)**
- "Explain like I'm new" toggle on any instruction step
- Generates a personalized checklist: "You said you have an ESP32 but no printer → here's a print service link"
- Troubleshooting chat scoped to that project's code + common fail reports

**Invisible everywhere else**
- No AI-generated project spam — AI assists humans, never posts
- Common questions in comments auto-promote into official instruction steps (your idea — implement it)

- If a project can't state those clearly, it doesn't get the 🟢 badge
- That badge becomes your brand promise — the thing people trust and share

# Storage: R2 vs S3 (for STLs, images, videos)

**Cloudflare R2 — my pick**
1. Free tier: 10GB storage, unlimited egress (downloads cost $0)
2. Egress is THE killer — STL/video downloads are your whole site; S3 charges ~$0.09/GB for that
3. S3-compatible API — if you learn R2, you know S3; switching later is trivial
4. Con: need a Cloudflare account (free, 5 min)
5. Con: no built-in image resizing on free tier (workable)

**AWS S3**
1. Free tier: 5GB for only 12 months, then pay
2. Egress fees will quietly bankrupt a file-sharing site — a viral post = a bill
3. More complex console/IAM setup
4. Only pick if you plan deep AWS usage later — you don't

**Also consider: Supabase (recommended alongside R2 or even instead, early)**
1. Free tier: Postgres database + auth + 1GB file storage in one
2. Solves your database AND accounts problem in one service
3. 1GB storage too small for STLs/videos long-term → use Supabase for data/accounts, R2 for files
4. Con: free tier pauses after 1 week inactivity (fine — it wakes on visit)

**Verdict:** Vercel (site) + Supabase (database + accounts) + R2 (files). All free, all scale, ~$0/month until real traffic.

# Accounts: Username-Only vs Email

**Username + password, optional recovery email (your plan) — do it**
1. Pro: zero friction, zero data liability, nothing to leak, no GDPR email headaches
2. Pro: matches your privacy-friendly brand — differentiator vs everyone
3. Con: forgot password + no recovery email = account permanently gone (show a big warning at signup)
4. Con: no email = no way to notify "someone commented on your post" (acceptable trade)
5. Con: slightly more impersonation risk (no verification) — mitigated by it not mattering much for a maker site

**Security notes (numbers)**
1. Supabase Auth handles hashing/sessions for you — never roll your own
2. Rate-limit login attempts (Supabase does this)
3. Don't store anything sensitive — no emails required means your worst-case breach is usernames + hashed passwords + robot files. Low stakes by design. That's a feature.

# Post-Without-Account (post password)

1. Pro: lowest possible friction — a teenager posts in 2 minutes, no signup wall
2. Pro: the "post password" (a random edit token) is simple and proven (like old pastebin)
3. Con: lost token = orphaned post forever (you have admin delete, so fine)
4. Con: anonymous posts need slightly stricter moderation defaults
5. Implementation ease: EASY — it's just a random string stored with the post, checked on edit
6. Suggestion: show the token ONCE with a big "copy this now" + offer "claim this post later by making an account" (token converts to ownership) — easy to add later

# The "Not a Bot" Test

1. Your Brave-style drag-to-dark-spot: charming but custom = you're building and maintaining anti-bot tech solo. Bots beat naive versions fast
2. Better: Cloudflare Turnstile — free, invisible (no puzzle for most humans), no data collection, no Google
3. You're getting Cloudflare anyway for R2 — Turnstile is 10 lines of integration
4. Compromise: launch with Turnstile, build your cute drag-puzzle later as a brand touch
5. "Once per first post then never again": store a verified flag on account/browser. EASY
6. Numbers: Turnstile = ~1 hour to add. Custom puzzle = days + ongoing arms race

# Post Structure (your spec, refined)

**Fields**
1. Name, short description
2. Prompted questions: "What problem does this solve?", "What quirks/gotchas should builders know?", "What was the hardest part?" — each optional, plain text
3. STL upload, code upload
4. Hardware checkboxes (auto-suggested from code scan, editable, custom entries with optional buy-links)
5. Instructions text box (AI-suggest button = placeholder for now — just an empty function you wire up later)
6. Video/photo upload

**Code-scan parts bot (no AI needed)**
1. It's regex/keyword matching: `Servo.h` → servo, `AccelStepper` → stepper + driver, `Adafruit_NeoPixel` → LED strip, pin definitions → wiring hints
2. Difficulty: EASY-MEDIUM — a lookup table of ~50 library names → parts. Genuinely no AI required
3. This is your best effort-to-wow ratio feature. Prioritize it

# Versioning (GitHub-style layers)

1. Simplest good version: every edit saves a full snapshot row; post shows latest; "history" tab lists old versions
2. Pro: dead simple, no diff logic, storage is cheap (text is tiny)
3. Con: big files re-uploaded each version eat R2 space — dedupe by file hash later
4. Difficulty: EASY for text fields, MEDIUM for files
5. Skip actual git-style diffs — snapshot model gives 95% of value for 5% of work
